# Curriculum / Course Outline Template

Use this to turn a diagnosed gap (from the TNA) into a course or multi-course curriculum a facilitator or vendor could actually build. The discipline that makes it work is **alignment**: every module traces to a measurable objective, and every objective is assessed. Nothing taught that isn't assessed; nothing assessed that wasn't taught.

> Fill only what's grounded. For proprietary or company-specific content (internal systems, product detail, house process), leave a clearly-marked `[CLIENT TO SUPPLY]` placeholder rather than inventing it. Keep the outline itself in `training-development/`; any roster or scores belong in the confidential zone.

## Audience

- **Who:** role(s), level, function.
- **Size / cohort:** number of learners; single cohort or ongoing intake.
- **Context:** on-site / hybrid / remote; time-available constraints; language/accessibility needs.
- **Entry assumptions:** what learners already know coming in.

## Prerequisites

- Prior courses, experience, access, or certifications required before starting. State "none" if open-entry.

## Learning objectives (measurable)

State what the learner will be able to **do** after the course — observable and measurable. Use **action verb + condition + standard**; avoid "understand", "know", "be aware of".

- Good: *"Given a customer refund request, process it in the POS system following the refund policy with no policy exceptions unapproved."*
- Weak: *"Understand the refund process."*

## Activities & practice

Adults learn by doing — pair content with practice, especially for apply-level objectives.
- Techniques: discussion, worked example, hands-on practice/simulation, role-play, case study, scenario, job aid walkthrough, group exercise, on-the-job task.
- Match the activity to the objective's level: recall → quiz/flashcard; apply → practice/simulation/role-play; analyze/evaluate → case study/scenario/decision exercise.
- Build in feedback — practice without feedback doesn't build skill.

## Assessment

How you'll confirm each objective was met.
- **Method:** knowledge check / practical demonstration / project / observation checklist / scenario judgment / certification exam.
- **Mapping:** each objective → the item(s) that assess it (reuse the O-numbers).
- **Pass standard:** the bar that counts as competent; for compliance, note that a required pass mark/attestation may be jurisdiction-specific — verify.
- **Records:** where completion/scores are logged (confidential zone) — never inline in this shareable outline.

## Logistics

- **Total duration:** and session structure (single session / multi-session / self-paced window).
- **Delivery mode:** instructor-led (in-person / virtual) / e-learning self-paced / blended / on-the-job coaching / cohort program / microlearning — with the reason it fits the objectives, audience, and `company.remote_policy`/locations.
- **Facilitator / instructor:** internal SME, external vendor, or self-service.
- **Materials:** slides, workbook, job aids, LMS content, environment/tools, `[CLIENT TO SUPPLY]` proprietary content — with brand styling from `company.design_file` where produced.
- **Accessibility:** captions, transcripts, accommodations, readable materials.

## Evaluation plan

Decide up front how you'll know it worked, using a levels model (e.g. Kirkpatrick). Not every course needs all four levels, but every course measures at least learning against its objectives.

| Level | Measures | Method | Applies to this course? |
|---|---|---|---|
| 1 — Reaction | Did learners find it relevant/useful? | Post-course survey | usually yes |
| 2 — Learning | Did they gain the knowledge/skill? | Pre/post assessment vs. objectives | **always** |
| 3 — Behavior | Are they doing it on the job? | Manager observation, follow-up, work samples (30–90 days) | for apply-level skills |
| 4 — Results | Did the business result move? | Metric tied to the TNA gap | for high-impact/strategic training |

- **Success criteria:** the specific numbers/observations that count as success at each level used.
- **Follow-up owner & timing:** who checks level 3/4 and when.
- **Iterate:** how evaluation feedback loops back to revise the course.
