# New-Hire Onboarding Checklist

A phase-organized checklist for taking an accepted offer to a fully ramped hire. Adapt to the role, level, employment type, and work arrangement (on-site / hybrid / remote). Each item names a suggested **owner** — reassign to match the client's actual roles. Check items off in the per-hire folder (`onboarding/<employee-slug>/`); keep any completed form that carries PII in the confidential records zone, not here.

> **Required new-hire forms vary by jurisdiction.** The paperwork rows below are generic. Which employment-eligibility / work-authorization, tax-withholding, new-hire-reporting, and mandatory-notice forms actually apply — and their deadlines and retention periods — depend on the hire's work location. Scope them to `legal.jurisdictions`, name the jurisdiction, and **verify with counsel**. Do not treat a specific form name or ID as fact for a jurisdiction you cannot confirm.

Owner key: **HR** = HR/People · **MGR** = hiring manager · **IT** = IT/systems · **FAC** = facilities/ops · **NEW** = the new hire · **BUD** = onboarding buddy/mentor.

## Phase 1 — Pre-boarding (from offer acceptance to day 1)

### Paperwork & compliance
| Done | Task | Owner |
|---|---|---|
| ☐ | Signed offer / acceptance filed (confidential zone) | HR |
| ☐ | Background/reference checks cleared where applicable (jurisdiction-scoped — verify) | HR |
| ☐ | Employment-eligibility / work-authorization verification arranged *(jurisdiction-specific — verify)* | HR |
| ☐ | Tax-withholding form(s) issued *(jurisdiction-specific — verify)* | HR / NEW |
| ☐ | Direct-deposit / payroll setup form issued | HR / NEW |
| ☐ | Emergency-contact form issued | HR / NEW |
| ☐ | Benefits enrollment packet issued with enrollment window noted | HR / NEW |
| ☐ | New-hire reporting / mandatory notices prepared *(jurisdiction-specific — verify)* | HR |
| ☐ | Employee record created in HRIS / files (PII in confidential zone) | HR |

### Equipment & system access (requests only — a human provisions)
| Done | Task | Owner |
|---|---|---|
| ☐ | Hardware requested (laptop, phone, peripherals, role-specific gear) | IT / MGR |
| ☐ | Core accounts requested (email, SSO, directory, calendar) | IT |
| ☐ | Role-specific tools & system access requested (least-privilege) | IT / MGR |
| ☐ | Physical access requested (badge, keys, parking) or remote-access kit | FAC / IT |
| ☐ | Software licenses / seats assigned | IT |

### Welcome & logistics
| Done | Task | Owner |
|---|---|---|
| ☐ | Pre-start welcome note sent (first-day when/where, what to bring, dress, parking/remote link, who greets them) | HR / MGR |
| ☐ | Day-1 schedule drafted and shared | MGR |
| ☐ | Workspace / remote kit prepared | FAC / IT |
| ☐ | Onboarding buddy/mentor assigned and briefed | MGR |
| ☐ | Team informed of start date and role | MGR |

## Phase 2 — Day 1

| Done | Task | Owner |
|---|---|---|
| ☐ | Personal greeting and workspace / remote setup confirmed working | MGR / BUD |
| ☐ | Any remaining paperwork completed *(jurisdiction-specific items — verify)* | HR / NEW |
| ☐ | Accounts and access confirmed live (login test) | IT / NEW |
| ☐ | Company & culture intro (grounded in ABOUT.md) | HR / MGR |
| ☐ | Core-policy walk-through **using the handbook from `policies/`** | HR |
| ☐ | **Handbook acknowledgment signed** (matches current handbook version) | HR / NEW |
| ☐ | Safety / emergency basics covered (coordinate `health-safety-wellness/` if warranted) | HR / FAC |
| ☐ | Manager 1:1 — role expectations, first-week plan, how we work | MGR |
| ☐ | Onboarding buddy introduction | BUD |
| ☐ | 30/60/90 plan shared and walked through | MGR |

## Phase 3 — Week 1

| Done | Task | Owner |
|---|---|---|
| ☐ | Team introductions and key cross-functional partners met | MGR / BUD |
| ☐ | Tools and systems orientation (hands-on) | MGR / IT |
| ☐ | Mandatory / compliance training assigned via `training-development/` | HR |
| ☐ | Role-specific training plan started | MGR |
| ☐ | First small, achievable task assigned (early win) | MGR |
| ☐ | Benefits enrollment completed within window | HR / NEW |
| ☐ | End-of-week check-in (manager + buddy) | MGR / BUD |
| ☐ | Remote/hybrid social integration touch scheduled | MGR |

## Phase 4 — First 90 days

| Done | Task | Owner |
|---|---|---|
| ☐ | 30-day check-in: onboarding experience, questions, early goals on track | MGR / HR |
| ☐ | 30/60/90 milestones reviewed and adjusted | MGR |
| ☐ | Required training on track / completed | HR / NEW |
| ☐ | 60-day check-in: ramping, feedback both directions | MGR |
| ☐ | Probation/introductory-period review if applicable *(jurisdiction/policy-specific — verify)* | MGR / HR |
| ☐ | 90-day check-in: "fully ramped" assessment vs. plan | MGR / HR |
| ☐ | Onboarding feedback collected to improve the process | HR |
| ☐ | Transition into the regular performance-management cycle | MGR / HR |
